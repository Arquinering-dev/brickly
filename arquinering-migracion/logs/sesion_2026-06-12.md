## Sesión 2026-06-12 — Dashboard web de control de gestión

### Objetivo
Construir un dashboard web ejecutivo que convierta los Resúmenes de Obra v8 (18 hojas) en
una vista legible para socios: portfolio multi-obra + detalle por obra. Datos leídos en vivo
de los .xlsx (sin base de datos), stack liviano, agregar obra = soltar archivo + declararla.

### Completado
- [x] **Mapa de datos** de las 7 hojas que alimentan el dashboard (0_CONFIG, 3_Dashboard,
      3_Control_Ppto, 3_Cash_Flow, 2_Certificaciones, 2_Subcontratos, 3_Control_Jornales),
      con todas las diferencias estructurales GDR↔CH documentadas.
- [x] **Arquitectura**: Flask + openpyxl (data_only) + frontend vanilla + Chart.js vendoreado.
      Carpeta `dashboard/`. Lectura **anclada por texto** (no coordenadas) por la deriva entre obras.
- [x] **Capa 1 — `reader/read_obra.py`**: contrato JSON completo, probado contra ambos archivos.
- [x] **Capa 2 — backend Flask** (`app.py`): `/api/obras`, `/api/obras/<code>`, vistas, 404.
- [x] **Capa 3 — vista portfolio** (cards comparativas con semáforo global).
- [x] **Capa 4 — vista detalle** (resumen ejecutivo, control ppto + avance etapa, cash flow chart,
      certificaciones cert→factura→cobro + apertura fiscal, subcontratos, jornales).
- [x] Verificación visual por screenshots headless (Chrome): ambas vistas renderizan OK.

### Decisiones tomadas (confirmadas por Pedro)
- **3_Dashboard / 0_CONFIG-calculados descartados como fuente** (layout inconsistente entre obras,
  CH con valores rotos). Todos los KPIs se derivan de las hojas de control tabulares.
- **Excel = fuente de verdad**: el dashboard lee el último guardado (no recalcula fórmulas).
- **Avance físico** = etapa ponderado (certif/ppto venta): 46,7% GDR · 9,5% CH. Consistente entre obras.
- **Cobrado** = ingresos acumulados del Cash Flow (cobros desde 2_Gastos): $660,6M GDR (= dashboard v8).
- **Semáforos recalculados** en el reader (los glifos guardados son inconsistentes entre obras).
- **Backend**: Flask + PyYAML (instalados en esta sesión).
